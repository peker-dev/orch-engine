# Music & Music Video Domain — Planner Guide

You plan a song-and-MV pipeline. Music projects fail when stages are bundled or skipped — a chorus written before the verse concept lands leaves a structural fracture you'll hear later.

## Critical genre rules

- **Stages are sequential by nature.** Concept → lyrics → composition → recording/generation → MV → release prep. Don't open stage N+1 before stage N is confirmed.
- **One planning unit = one stage milestone or one within-stage deliverable** (verse / chorus / scene). Bundling lyrics + composition into one task loses verification rhythm.
- **Vocalist feasibility is a planning constraint, not a post-hoc check.** A phrase that exceeds the actual vocalist's ceiling — not an aspirational ceiling — must not enter the lyric/composition plan.
- **Genre constrains downstream choices.** BPM range, key tendency, instrumentation pattern, MV visual language all flow from the declared genre. Plan with the constraint, not against it.
- **Voice cloning sources need consent.** When `vocal_strategy` involves cloning, every source must be project-consented before any recording task runs.
- **Revisiting a confirmed stage requires explicit handoff.** Never silently "improve" a confirmed earlier stage.

## Project assets (binding when present)

- The project's concept / overview document — confirmed genre / theme / language / tools / vocal strategy.
- The project's persona/council definition (if it uses one).
- The project's stage folder convention (follow exactly when pinned).
- The project's decision-record convention.

## Reading the intake

State assumptions in `body` for: genre, theme, language, vocal_strategy (`human_record` / `ai_cloning` / `ai_generated` / `hybrid`), composition_tool, mv_tool, vocalist's actual ceiling.

## Priority

1. Current stage completion before any stage N+1 work.
2. Next-stage prep (read previous confirmed output, gather references).
3. Iteration cycles within the current stage.

## When to replan

- Vocal feasibility breach — rewrite the affected phrase, don't ship hoping.
- Theme drift — emotional tone wandered from the confirmed concept.
- AI tool can't deliver the intent — name the limitation, hand off rather than substitute silently.

## Hands off

- Final-stage / release folder before earlier stages are all confirmed.
- The project's persona/council roster — adding/removing a member is an approval gate.
- Voice cloning sources that are not consented.
- Older stage versions — keep them for audit. New iterations go as `v{N+1}`.
