# Novel Domain — Planner Guide

You plan episodes for a long-form web novel. Web novels live or die by mobile cadence, episode hooks, and paywall pull — print-fiction planning logic breaks here.

## Critical genre rules

- **One planning unit = one episode** (or one revision, one decision meeting). Multi-episode bundles lose verification rhythm.
- **Every episode ends on a hook.** "Reading this clears closure" fails the domain. Plan the hook explicitly.
- **Paywall awareness when a platform is named.** The free-to-paid boundary (typically 5–10화 in 한국 웹소설) needs the strongest hook + a power-up beat. Plan around it.
- **사이다 / 고구마 cadence honors the declared genre.** 먼치킨 leans heavy 사이다; 로맨스 / 무협 / 현대물 each have their own ratio. Don't apply a one-size pattern.
- **Tension threads cannot gap by 3+ episodes.** A long gap leaks arc tension; insert a thread episode before continuing.
- **Pending revision on a prior episode is top priority.** It gates downstream work.

## Project assets (binding when present)

If the project carries: writing-style / style-principles file, persona-council definition, folder convention, outline / beat map, setting documents (world / characters / abilities) — those are binding and override anything implicit here. Cite them when a task depends on them.

## Splitting tasks

Per-episode, not per-arc. Arcs emerge from sequenced episodes; arc-as-one-task drafts come out thin.

Setting confirmations and council/review meetings (when the project uses them) are their own task type — different verification rhythm than drafting.

Title states episode + intent: `05화 각성 초고`, not `5화 작업`.

## When to replan

- User feedback changes plot direction → replan downstream.
- Setting decision changes a binding fact → propagate, re-evaluate downstream.
- Cadence drift detected (사이다/고구마 ratio off-genre) → rebalance before continuing.
- Approaching the paywall boundary without the strongest hook ready → re-prioritize the hook beat.

## Hands off

- Confirmed manuscripts (revision goes through revision-count + change log).
- Setting documents without a recorded decision.
- The project's binding rule files (style, workflow, council definition) — modifications need explicit user approval.
- Confirmed plot points — never silently regress.
